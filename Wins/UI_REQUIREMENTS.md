# GlimpseMe UI 需求文档

> **目标读者**：设计 AI / UI 设计师
> **用途**：根据本文档设计 GlimpseMe 应用的视觉界面

---

## 一、产品定义

### 这是什么？

**GlimpseMe** 是一个"选择即标注"的个人数据收集工具。

用户在日常使用电脑时，看到感兴趣的内容（文字、代码、对话...），只需**选中它**并按下快捷键，系统会自动记录这段内容，同时弹出一个小浮窗让用户快速表达态度（喜欢/不喜欢/一般）并可选地添加想法。

### 核心理念

> **"复制"这个动作，本身就是一种标注。每一次框选，都是你在告诉世界——这个东西对我有意义。**

我们不只是做一个剪贴板工具，我们在帮用户构建一份**属于自己的数据档案**——记录你关注什么、喜欢什么、思考什么。这份数据未来可以让 AI 更懂你。

### 目标用户

- 知识工作者（研究员、写作者、开发者）
- 想要记录灵感和想法的人
- 希望 AI 更了解自己偏好的人

---

## 二、产品结构

GlimpseMe 有三个主要界面：

| 界面 | 触发方式 | 核心功能 |
|------|----------|----------|
| **1. 浮窗** | 快捷键 (Alt+Q) | 快速标注当前选中内容 |
| **2. 主界面** | 点击托盘图标 | 查看所有收集的内容 |
| **3. 设置页** | 主界面内进入 | 配置全选行为、快捷键等 |

---

## 三、界面一：浮窗 (Floating Window)

### 3.1 什么时候出现？

用户在**任意应用**中选中内容后，按下快捷键（默认 Alt+Q），浮窗出现在鼠标位置。

### 3.2 界面元素

```
┌─────────────────────────────────────────┐
│                                         │
│   👍        😐        👎               │
│  喜欢      一般      不喜欢             │
│                                         │
│  ┌─────────────────────────────────┐   │
│  │ 添加想法...                     │   │
│  └─────────────────────────────────┘   │
│                                         │
│  ☐ 全选                                │
│                                         │
└─────────────────────────────────────────┘
```

### 3.3 元素说明

| 元素 | 必要性 | 为什么需要 |
|------|--------|-----------|
| **反应按钮 (3个)** | 必需 | 最快的标注方式，一键表达态度 |
| **文本输入框** | 可选 | 用户可能想记录为什么喜欢/不喜欢 |
| **全选复选框** | 可选 | 有时候想保存整页内容作为上下文 |

### 3.4 交互行为

- **点击反应按钮**：标记态度（单选）
- **输入文字**：记录想法
- **勾选全选**：自动执行 Ctrl+A 选中整页
- **按 Enter**：保存并关闭浮窗
- **按 Esc**：取消并关闭浮窗
- **点击浮窗外部**：关闭浮窗（可配置是否保存）

### 3.5 设计原则

1. **极简**：不超过 5 个交互元素，10 秒内完成操作
2. **轻盈**：窗口小巧，不遮挡内容，有微妙阴影感
3. **不打扰**：暗色调（可选明/暗模式），不刺眼
4. **快速响应**：按下快捷键到显示 < 100ms

### 3.6 视觉资产需求

| 资产 | 规格 | 说明 |
|------|------|------|
| 喜欢图标 | 24x24 SVG | 可用 emoji 或自定义图标 |
| 一般图标 | 24x24 SVG | - |
| 不喜欢图标 | 24x24 SVG | - |

## 四、界面二：主界面 (Main UI)

### 4.1 入口

用户点击系统托盘图标，打开主界面。

### 4.2 功能

- **查看历史记录**：所有收集的内容
- **筛选/搜索**：按态度、时间、来源筛选
- **查看详情**：点击条目查看上下文

### 4.4 信息层级

**第一层（列表视图）**：
- 态度图标 (👍/😐/👎)
- 内容预览（截断显示）
- 来源（应用名 + 页面/联系人）
- 时间

**第二层（详情视图）**：点击条目展开
- 完整选中内容（高亮）
- 用户添加的想法
- 上下文/背景内容（如果有全选）
- 完整来源信息

### 4.5 元素说明

| 元素 | 必要性 | 为什么需要 |
|------|--------|-----------|
| **态度图标** | 必需 | 快速识别这条记录的情感色彩 |
| **内容预览** | 必需 | 知道记录了什么 |
| **来源标签** | 必需 | 知道内容来自哪里 |
| **时间戳** | 必需 | 时序感，方便回溯 |
| **搜索框** | 必需 | 快速找到特定内容 |
| **筛选器** | 推荐 | 按态度/来源/时间筛选 |

### 4.6 设计原则

1. **信息密度适中**：每屏显示 5-8 条记录
2. **渐进式披露**：默认简略，点击展开详情
3. **来源可视化**：不同来源有不同颜色/图标
4. **情感色彩**：喜欢=暖色调，不喜欢=冷色调

### 4.7 视觉资产需求

| 资产 | 规格 | 说明 |
|------|------|------|
| 应用图标 | 16x16, 32x32, 48x48, 256x256 | 托盘图标 + 主窗口图标 |
| 来源图标集 | 16x16 SVG | 浏览器、微信、Notion、VSCode 等 |
| 空状态插图 | - | 当没有记录时显示 |

## 五、界面三：设置页 (Settings)

### 5.1 功能

| 设置项 | 说明 |
|--------|------|
| **快捷键** | 自定义触发浮窗的快捷键 |
| **全选行为** | 默认勾选/不勾选；选取范围（全部/前后 50%） |
| **外观** | 明/暗模式 |
| **存储位置** | 数据保存路径 |
| **启动** | 是否开机自启动 |

### 5.2 布局

简洁的分组列表，每个设置项一行。

## 六、整体设计风格建议

### 6.1 色彩

| 元素 | 明模式 | 暗模式 |
|------|--------|--------|
| 背景 | #FFFFFF | #1E1E1E |
| 卡片 | #F5F5F5 | #2D2D2D |
| 主色 | #4CAF50 (绿) | - |
| 喜欢 | #4CAF50 (绿) | - |
| 一般 | #9E9E9E (灰) | - |
| 不喜欢 | #F44336 (红) | - |

### 6.2 字体

- **标题**：系统 UI 字体（Windows: Segoe UI）
- **正文**：等宽字体 for 代码内容
- **中文**：Microsoft YaHei

### 6.3 动效

- 浮窗出现：淡入 + 轻微缩放 (100ms)
- 按钮点击：微弹跳反馈
- 列表加载：骨架屏过渡

## 附录：产品价值观

> **简单 > 功能多**：每个功能都有明确的使用场景
> **隐私优先**：数据本地存储，用户完全掌控
> **为 AI 准备**：数据结构化，未来可作为 AI 了解用户的素材
